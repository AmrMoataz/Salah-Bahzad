# Native App · A2 — APP stream (core/secure_surface: the real per-OS screen-capture black-out)

> Status: **Planned — not yet built** · Created 2026-06-26 · The **app half** of phase **A2** in `docs/IMPLEMENTATION-PLAN-native-app.md` (§7-A2, lines 201–209; native-call matrix §4.2 lines 114–123). Builds the **NEW `app/lib/core/secure_surface/`** plugin that replaces A1's static **"Screen capture blocked"** banner with the **real OS-level black-out**: Android `FLAG_SECURE`, Windows `SetWindowDisplayAffinity(WDA_EXCLUDEFROMCAPTURE)`, macOS `NSWindow.sharingType = .none`, iOS `UIScreen.isCaptured` KVO → blank/pause + `userDidTakeScreenshot` → log/flag. Protection toggles **ON before the first frame** and **OFF on leaving the player** — filling the A1 capture-flag stub at `player_page.dart:49`. Where the black-out **can't be guaranteed** (e.g. Windows < 2004), the app **detects the capability and warns + refuses** protected playback (`NFR-APP-COMPAT-002` — fail safe, not silent), and the banner is driven from the **real** protection state. **A2 has NO backend stream and NO contract change** (`Backend: none`, master line 205) — there is no DTO, no route, and no `docs/contracts/*` surface; the only "contract" is the §4.2 native-call matrix. **If a native API in §4.2 must move, change master plan §4.2 first.**

## Conventions (mirror `app/CLAUDE.md` + `frontend/CLAUDE.md` discipline)
- Latest stable Flutter; `flutter analyze` **clean** (zero issues); `dart format` enforced; null-safe.
- **Riverpod** (`Notifier` + `NotifierProvider`, no codegen) + a thin platform seam. The native channel is wrapped behind a fakeable Dart facade — **the real `MethodChannel`/`EventChannel` is never constructed in `flutter test`** (`NFR-APP-REL-003`).
- **Native belongs in `core/` + the page, never the pure `…View`** (`app/CLAUDE.md` view↔page split): the OS toggle is hosted by `player_page.dart` (the `ConsumerStatefulWidget`); `player_view.dart` stays a deterministic, golden-tested `…View` that only **renders** the protection *state*, never *calls* a channel.
- **One responsive widget tree** — the banner reflows inside the existing player Stack; breakpoints `compact < 560 / medium 560–1024 / expanded ≥ 1024`. Never a phone build vs a desktop build.
- **Design tokens only** — `SbColors`/`SbMotion`/`SbFonts`/`SbRadii`, never an inline hex/size.
- **Security hygiene (this *is* the protection mechanism):**
  - **No unprotected window of exposure** (`NFR-APP-CAP-006`): the toggle fires **synchronously in `initState`** (before the route paints and well before the engine opens) — never after the first frame.
  - **Fail safe, not silent** (`NFR-APP-COMPAT-002`): on a platform/version where the black-out can't be guaranteed, **warn + refuse** protected playback — **never** open the engine unprotected. The decision is driven by the **real** `SecureSurfaceStatus`, not by an assumption about the OS.
  - The facade **never logs** tokens/handoffs/PII (it has none) and **never** disables TLS (it touches no network). It is a pure platform call.
  - iOS still-screenshots are an **honest accepted gap** (`NFR-APP-CAP-005`): detect + flag, **do not** claim a block — the visible watermark (`FR-APP-VID-003`, A1) is the deterrent. FairPlay is **deferred, not v1** (master §2; out of scope below).
- **Reuse A1, do not rebuild** — consume these exact seams, **do not widen them:**
  - The toggle host points: `player_page.dart` `initState` (`:34`) for **ON** before `start(request)` (`:50`), the stub comment at `:49` to replace, and `dispose()` (`:55-59`) for **OFF** alongside `_applyFullscreen(false)` (`:57`). The first frame is produced when the engine opens (`player_controller.dart:89` `_engine.open`) — that is the deadline for ON.
  - The static banner to make truthful: `capture_blocked_banner.dart:11-41` (today a parameterless `StatelessWidget`) and its mount site `player_view.dart:92-97`.
  - The per-OS capability seam: `core/platform/app_platform.dart` — `AppTarget` (`:8-17`), `isDesktop` (`:34`), `isWindows` (`:39`). An A2 capability getter belongs here.
  - The fakeable-facade + per-platform-branch precedent: `VideoEngine` (`core/playback/video_engine.dart`, faked in `test/support/playback_fakes.dart:154-229`, provider `providers.dart:131-135` `.autoDispose`) and `GoogleCredentialSource` (`features/auth/google/google_credential_source.dart`, picked per-platform in `providers.dart:148-157` via `if (platform.isWindows) … else …`). **A2 mirrors exactly this shape.**
  - The runner registration spots: Android `MainActivity.kt:5` (bare `FlutterActivity()`); Windows `flutter_window.cpp:27` (`RegisterPlugins(...)`, HWND from `GetHandle()` per `win32_window.h:50`); macOS `MainFlutterWindow.swift:11` (`RegisterGeneratedPlugins(registry:)`); iOS `AppDelegate.swift:13-15` (`didInitializeImplicitFlutterEngine` → `GeneratedPluginRegistrant.register`).

## Steps

### F1 — Windows black-out + COMPAT capability **spike** (`app/windows/runner/flutter_window.cpp`) ⚠️ DO THIS FIRST
- **Prove the real exclusion (`NFR-APP-CAP-002`):** in `flutter_window.cpp` (after `RegisterPlugins` at `:27`), call `SetWindowDisplayAffinity(GetHandle(), WDA_EXCLUDEFROMCAPTURE)` (user32, HWND from `win32_window.h:50`) and confirm with a **real screen recorder** (OBS) + Snipping Tool that the window is **excluded from capture** on this Win10/11 box. Reset to `WDA_NONE` to confirm the off-path restores normal capture.
- **Prove the capability gate (`NFR-APP-COMPAT-001/002`):** `WDA_EXCLUDEFROMCAPTURE` requires **Windows 10 2004 / build 19041**. The spike must implement an **OS-build check** that, on a build **< 19041**, returns the native "unsupported" result → the Dart facade maps it to `SecureSurfaceStatus.unsupported` → the player **warns + refuses** (F7). **Resolve in this spike:** whether the legacy **`WDA_MONITOR`** flag (the pre-2004 display-affinity value — **not** documented in the master, whose §4.2 line 119 specifies only `WDA_EXCLUDEFROMCAPTURE`/`WDA_NONE`) is an acceptable *guaranteed* black-out on pre-2004 builds, or whether pre-2004 is simply **refused** — the master §2 (line 57) / COMPAT-002 default is **refuse** (treat pre-2004 as unguaranteed). Record the decision.
- **Output of the spike:** a one-paragraph note in the A2 wiring doc recording (a) that `WDA_EXCLUDEFROMCAPTURE` visibly blacks out capture here, and (b) the chosen pre-2004 behaviour (refuse vs `WDA_MONITOR`). Only after this passes do F2–F7 proceed. (Android `FLAG_SECURE` is **not** a risk — it predates every floor; see F4.)

### F2 — The Dart facade + COMPAT status (`app/lib/core/secure_surface/`)
- **CREATE `app/lib/core/secure_surface/secure_surface.dart`** — the abstract seam (mirror `VideoEngine`):
  - `abstract class SecureSurface { Future<SecureSurfaceStatus> enable(); Future<void> disable(); }`
  - `enum SecureSurfaceStatus { protected, unsupported, off }` — `protected` = the OS black-out is active; `unsupported` = capability absent (Windows < 2004 / iOS best-effort) → the COMPAT-002 refuse path; `off` = not engaged. **This enum drives both the banner (F6) and the refuse gate (F7).**
- **CREATE `app/lib/core/secure_surface/method_channel_secure_surface.dart`** — the prod impl over a single `const MethodChannel('salah_bahzad/secure_surface')` (§3.1: "one `MethodChannel` per desktop platform + an iOS `EventChannel`"). Methods: `enable` / `disable` / `isSupported`. A native **"unsupported"** reply (or a missing handler) maps to `SecureSurfaceStatus.unsupported` — never throws into the player. (iOS also wires the `EventChannel('salah_bahzad/secure_surface/events')` stream for `isCaptured`/`userDidTakeScreenshot`; see F4/F5.)
- **CREATE `app/lib/core/secure_surface/noop_secure_surface.dart`** — returns `unsupported`/`off`; the **safe default** for any platform left unwired so the app never crashes over protection. (Mirrors how `NoopVideoEngine`-style fakes keep tests off native.)
- **No new pub dependency** — the channel is registered in each runner, exactly as `window_manager`/`firebase` are runner-registered (`app/CLAUDE.md`). **No `pubspec.yaml` change.**

### F3 — Provider wiring (`app/lib/app/providers.dart`)
- **CREATE `secureSurfaceProvider`** next to `videoEngineProvider` (`providers.dart:131-135`), branching on `appPlatformProvider` **exactly like `googleCredentialSourceProvider` (`providers.dart:148-157`)**: a real `MethodChannelSecureSurface` on the four targets, falling back to `NoopSecureSurface` for any unhandled case. Overridable with a `FakeSecureSurface` in tests.
- **CREATE `secureSurfaceStatusProvider`** — a tiny `Notifier<SecureSurfaceStatus>` (default `off`) the page writes after `enable()` resolves, and the view reads to render the banner + drive the refuse gate. Mirror the existing `Notifier`/`NotifierProvider` style in `providers.dart`.

### F5 — Player page hook points (`app/lib/features/player/player_page.dart`)
- **ON (`NFR-APP-CAP-006`/`FR-APP-CAP-003`):** in `initState` (`:34`, right after `super.initState()`, **before** the post-frame `start(request)` at `:50`) call `ref.read(secureSurfaceProvider).enable()` and write the returned `SecureSurfaceStatus` into `secureSurfaceStatusProvider`. **Replace the stub comment at `:49`** (`// A2 will toggle the OS capture black-out here (no-op in A1).`). Firing in `initState` guarantees protection precedes the route paint and the engine open (`player_controller.dart:89`) — **no unprotected window**.
- **OFF (`FR-APP-CAP-003` "disabled only when leaving the player"):** in `dispose()` (`:55-59`) call `ref.read(secureSurfaceProvider).disable()` alongside the existing `_applyFullscreen(false)` (`:57`), wrapped in the **same crash-proof `try/catch`** style (`:70-72`) so teardown never throws.
- **iOS reactive blank/pause (`FR-APP-CAP-002`, `NFR-APP-CAP-004`):** subscribe to the `EventChannel` stream (F4) in the page; on an `isCaptured` event, pause/blank via the existing `playerControllerProvider` (`togglePlay`/state), resuming when capture stops. **Keep the subscription off the pure view** (page-only). On non-iOS platforms the stream is empty/absent — a no-op. (Stream behaviour is **`Mac-required`** to verify.)
- **Keeps native off the pure `…View`** per the view↔page split — the page hosts every channel call; the view only renders state.

### F6 — Banner driven by real protection state (`app/lib/features/player/widgets/capture_blocked_banner.dart`, `player_view.dart`)

### F7 — COMPAT-002 warn + refuse gate (`app/lib/features/player/player_page.dart` / `player_controller.dart`)
- **Refuse, never play unprotected (`NFR-APP-COMPAT-002`):** when `enable()` resolves to `SecureSurfaceStatus.unsupported` on a platform where the black-out is **required** (Windows < 2004, and any desktop/Android target that reports unsupported), the page **must not** start protected playback — surface a **player-reachable refusal state** (reuse the A1 inline error path / `PlayerError` so the existing overlay renders it) with a clear "can't protect this lesson on this device" message + a non-playing route. **Do not** call `start(request)` / open the engine in this branch.
- **iOS is the one best-effort exception:** iOS reports best-effort (recording defeated via `isCaptured`, still-screenshots an accepted gap) — it **plays** with the **amber** banner + watermark, **not** refused (`NFR-APP-CAP-005`). Keep this branch explicit so iOS is not swept into the desktop refuse path.
- **Drive the banner from the same status** (F6) so the on-screen state and the play/refuse decision can never disagree.

## Exit criteria
- The NEW `core/secure_surface` facade exists with `SecureSurface` + `SecureSurfaceStatus`, branched per-platform in `providers.dart` (`NFR-APP-CAP-001..004`).
- Protection toggles **ON in `initState` before the first frame** and **OFF in `dispose`** — the A1 stub at `player_page.dart:49` is filled; unit/widget tests prove `enable==1` on mount / `disable` on leave (`FR-APP-CAP-003`, `NFR-APP-CAP-006`).
- **Android** `FLAG_SECURE` and **Windows** `WDA_EXCLUDEFROMCAPTURE` black out a real screenshot **and** a real screen-record (manual, on this box); the off-path restores normal capture (`NFR-APP-CAP-001/002`).
- `Mac-required`: **macOS** `sharingType=.none` and **iOS** `isCaptured` blank/pause + `userDidTakeScreenshot` flag are written and (on a Mac) build + manually verified (`NFR-APP-CAP-003/004/005`).

- The banner is driven by the **real** `SecureSurfaceStatus` (protected/unsupported), making the A1 reassurance banner truthful (`FR-APP-CAP-001`).
- `flutter analyze` clean; `flutter test` green (facade + COMPAT unit + page lifecycle + banner goldens at 3 widths); `flutter build windows --debug` and `flutter build apk --debug` succeed. (macOS/iOS builds are **`Mac-required`** and not part of this box's gate.)
